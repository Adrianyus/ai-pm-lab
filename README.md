# AI 产品需求助手（AI PM Assistant）

## 1. 项目背景
在产品需求评审过程中，PRD 经常存在需求遗漏、边界不清、逻辑冲突的问题，
依赖人工 review 成本高、效果不稳定。

## 2. 解决的问题
- 自动识别 PRD 中可能遗漏的功能点
- 提示潜在的业务/技术冲突
- 辅助 PM 进行需求拆解与补充

## 3. 产品设计
### 使用流程
1. 用户上传 PRD 文档
2. 系统进行文本切分与向量化
3. 基于 RAG 检索相关需求片段
4. LLM 生成结构化需求分析结果

### 关键设计决策
- 使用 RAG 而非微调，原因是需求文档差异大
- 输出结构化结果，便于 PM 直接使用

## 4. 技术方案
- LLM：Qwen / GPT-4
- 向量数据库：FAISS
- 框架：LlamaIndex
- 后端：Python + FastAPI

## 5. 示例效果
输入：一份电商 PRD  
输出：
- 潜在遗漏需求：3 项
- 风险提示：2 项
- 优化建议：5 条

## 6. 局限与改进方向
- 对非结构化 PRD 支持有限
- 暂不支持跨文档需求对齐
